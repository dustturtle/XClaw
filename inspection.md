# 真实环境验收脚本

按顺序执行即可，`是否通过` 建议填写 `通过 / 不通过 / 备注`。如果失败，顺手记录原始回复和发送时间，方便回查日志。

## 微信链路

| 编号 | 问题 | 预期能力 | 是否通过 |

| 1 | 打开邀请页并扫码绑定微信 | 页面能正常显示二维码；扫码后绑定状态变为 `confirmed` |  |
| 2 | 微信里发送：`你好，你能做什么？` | Bot 能收到消息并在合理时间内回复，不出现长时间无响应 |  |
| 3 | 微信里发送：`帮我再讲一个故事` | 如果回复超过约 0.5 秒，应先出现微信原生“输入中”；对于较慢任务，中间不应长时间空窗，最终仍要收到正式回复 |  |
| 4 | 微信里发送：`今天A股大盘怎么样` | 能返回上证、深证、创业板等指数概览；A股语义应是红涨绿跌，不要出现“绿色上涨” |  |

## 任务助理能力

| 编号 | 问题 | 预期能力 | 是否通过 |
|---|---|---|---|

| 24 | `帮我创建一个每天15:00执行的任务：给我一份A股大盘摘要` | 能创建 cron 周期任务，并返回成功提示 |  |
| 25 | `查看我的定时任务列表` | 能看到周期任务，且带有 cron 或下次执行时间信息 |  |
| 26 | `取消刚才那个每天15:00的大盘摘要任务` | 能根据任务 ID 或上下文取消定时任务，并返回取消成功提示 |  |

## 任务助理补充观察

- 定时任务是“仅创建成功”，还是“到点后真的执行并产生结果”。
- 一次性任务触发后，结果是否能主动回到原微信会话；如果没有，请记录是否有任何后台执行痕迹或延迟。
- 周期任务取消后，再次查看列表时应不再显示为活跃任务。

## 重点观察

- 微信回复是否稳定，有没有“消息发出后很久才回”。
- 微信“输入中”是否稳定：慢回复场景下是否出现，长回复场景下是否会持续到正式回复发出，快回复场景下是否不会无意义闪烁。
- A股回答里颜色语义是否正确：涨应偏“红”，跌应偏“绿”。
- 缺口分析是否准确：价格区间重叠不应被误判为缺口；个股和指数代码不要串台。
- 涉及外部数据源的能力如果失败，是否给出明确降级说明，而不是空白或报错堆栈。
- 任务助理是否只会“记住任务”，还是能稳定完成“创建 -> 查看 -> 触发 -> 取消”的闭环。
- 只有当入站消息本身携带 `message.context_token` 时，才会尝试展示微信原生“输入中”；如果消息可正常回复但没有 `context_token`，这次不会展示 typing。
